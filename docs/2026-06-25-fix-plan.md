## 详细方案

### 1. #15 修 `_proxy` buffer 阻塞（P0）

**根因总结**（已修正 pending-fixes 描述）：
- `_proxy` 在 job-creation 路径强制 `resp.read()` 读完整 body 才写 wfile（`portal/app.py:1235-1256`）
- 子应用 POST `/api/jobs` 入口（如 `seedance/app.py:1441`）已是异步 `threading.Thread(daemon=True).start()` + **立即返回** job_id，所以 _proxy 等 body 通常 ms 级
- 真正让用户"网站连不上"的是**浏览器 6 连接限制** + **listen backlog=5** + **每条慢请求多耗一个 Portal 线程**的叠加效应

**改造方案（B 方案：X-Job-Id 响应头）**：

1. **子应用层**（4 个 app.py）：所有 job 创建路径（`/api/jobs` POST 等）在响应里加 `X-Job-Id` header，body 保持原样不变：
   - `seedance/app.py` `do_POST` 写 job_id 时同步设 `self.send_header("X-Job-Id", job_id)`
   - `nano-banana/app.py` 同
   - `dreamina/app.py` 同
   - `volcengine-portrait/app.py` `/api/virtual/jobs` 与 `/api/real/jobs` 同

2. **Portal 层**（`portal/app.py:1172-1276` `_proxy`）：
   - 删除 `should_buffer = is_job or "application/json" in content_type` 中的 `is_job`，让 job 创建路径走 streaming 分支
   - streaming 分支在写 headers 前 `peek` `resp.getheader("X-Job-Id")`，调用 `tracker.register_job` 并继续 `shutil.copyfileobj` 流式回写
   - 保留原 `application/json` 走 buffer 的路径（响应小 + 前端要拿整 body）
   - **fallback**：如 `X-Job-Id` 缺失则记 warning，不影响下行

3. **listen backlog 增大**：在 `main()` 1346 创建 server 之后加 `server.socket.listen(64)`（默认 5，改 64 给突发并发余量）。

4. **请求体也不 buffer**（可选加分项）：把 `self.rfile.read(length)` 改成边读边 `conn.send(chunk)`。但这要求 `http.client.HTTPConnection` 支持流式 body — `conn.send(data)` 可以，但接口侵入大，**本次先不做**，仅做响应侧。

**改动行号清单**：
- `portal/app.py:1172-1276` `_proxy` 改 streaming 分支
- `portal/app.py:1346` 加 `server.socket.listen(64)`
- `seedance/app.py:1441` 附近的 do_POST job_id 返回处
- `nano-banana/app.py` 同模式
- `dreamina/app.py` 同模式（注意 `/api/text2image`/`/api/image2image`/`/api/text2video` 等多个 job 入口都要加）
- `volcengine-portrait/app.py` `/api/virtual/jobs` `/api/real/jobs`

**风险**：
- 子应用任一漏改 `X-Job-Id` → 统计丢失 + Portal warning 日志。修复：写一个测试 `curl -ki POST /seedance/api/jobs` 看 response header 有 `X-Job-Id`。
- 前端依赖 body 里 `job_id` 字段不变，不破坏。

### 2. #8 VPN 抢路由（P3）

`portal/app.py:61-87` `get_lan_ip`：

```python
PRIVATE_PREFIXES = ("192.168.", "10.")
# 172.16-31 是私有段；172.32-255 是公网，要逐一判断
def _is_172_private(ip):
    if not ip.startswith("172."):
        return False
    try:
        second = int(ip.split(".")[1])
        return 16 <= second <= 31
    except (ValueError, IndexError):
        return False

VPN_BLACKLIST_PREFIXES = ("240.", "100.64.", "100.65.", ..., "100.127.", "169.254.", "198.18.", "198.19.")
```

修改：
- ifconfig 解析路径：跳过 VPN_BLACKLIST，匹配 192.168/10/`_is_172_private`
- fallback `socket.connect(8.8.8.8)` 拿到 IP 后也走同样过滤；不通过则 fallback 到 `127.0.0.1` + 启动期 print 警告

**改动行号**：`portal/app.py:61-87` 完整重写 `get_lan_ip`，约 +20 行。

### 3. #10 孤儿端口加固（P3）

调研明确：
- `_kill_port_squatter`（350）已是"启动前无差别 kill 端口 listener"路径 — 这是**最佳兜底**
- 真正的孤儿来源是 `launchctl kickstart -k` 实测 SIGKILL，finally 跳过

加固：
- **`_kill_port_squatter` 加 retry**：调研报告评分最高的 B 方案已存在，但有 lsof timeout=5 单次。改成"如 kill 后再次 lsof 仍 LISTEN，retry 一次"
- **子进程组级 SIGKILL**：当前 `start_app` 用 `preexec_fn=os.setsid`（408），`shutdown` 用 `os.killpg(getpgid, SIGTERM)`（467）。但 SIGKILL 时这一步跳过。加 `atexit.register(manager.shutdown)`（线程已 daemon=True 不阻塞退出，atexit 是 Python 层 hook，不会救 SIGKILL，所以这一条作用有限）
- **真兜底**：portal 启动时（`main()`）在 `manager.start_all()` 之前先 `for port in [8787,8797,8888,8891]: _kill_port_squatter(port)`，无差别清残留（当前 start_app 已对每个 port 调用，但**写在主循环里**等于已经覆盖）。检查这一路径是否真生效——确认无误后无需改

**实际改动**：仅在 `_kill_port_squatter`（350-381）末尾加 retry（如 kill 后 sleep 0.5s 再 lsof，仍占则 sleep 1s 再 kill 一次），约 +10 行。

### 6. 导出接口（新功能）

**接口**：`GET /api/platform/stats/export?start=YYYY-MM-DD&end=YYYY-MM-DD`

- **格式：CSV + UTF-8 BOM**（Excel/WPS 双击直显中文，纯 stdlib `csv` 模块写）
- **列布局：宽格式（一行一用户）**，列定义：
  ```
  username,
  seedance_images, seedance_seconds,
  nano-banana_images, nano-banana_seconds,
  dreamina_images, dreamina_seconds,
  volcengine-portrait_images, volcengine-portrait_seconds,
  total_images, total_seconds
  ```
  - `images` 列对图片任务计数（int）
  - `seconds` 列对视频任务的 done × duration 累加（int 秒）
  - `total_*` 是该用户在区间内跨所有 app 的汇总
- 数据源：`UsageTracker._data["by_user"]` 在 `[start, end]` 区间，对每用户 × 每 app 把 images / seconds 在区间内求和
- 权限：admin 看所有人；非 admin 只看自己（沿用 `get_history` 的 `role/username` 过滤）
- 默认参数：start 缺省 = 7 天前，end 缺省 = today
- 文件名：`usage-{start}-{end}.csv`
- HTTP 头：`Content-Type: text/csv; charset=utf-8` + `Content-Disposition: attachment; filename="..."`

**后端改动**：
- `UsageTracker` 新增 `export_range(start, end, username, role) -> list[dict]`
- `portal/app.py:840-846` 加 `/api/platform/stats/export` 路由 → `_platform_stats_export(user)`
- handler 写 CSV/JSON 时手动 send_header `Content-Disposition: attachment; filename="..."`

**前端改动**：
- `portal/static/app.js` 在 `StatsApp()` 加 `exportFrom / exportTo / exporting` 字段、`exportRange()` 方法（fetch+Blob，按 MEMORY 的"自签 HTTPS 下载修复"）
- `portal/static/index.html` 在「按日期查看」section（L554 附近）加日期段输入 + 导出按钮
