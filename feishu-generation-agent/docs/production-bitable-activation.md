# 生产多维表格启用清单

生产表只用于读取任务。应用不会创建、更新或删除生产表中的任何记录、字段或进度；本地锁只保存在本机 SQLite 数据库。

1. 将飞书应用作为文档应用添加到生产多维表格，并授予读取权限。
2. 确认应用能读取每条需求附件对应的飞书文档，并可下载其中的图片素材。
3. 仅在本机 `.env` 中配置 `LARK_PRODUCTION_BITABLE_URL`、`LARK_PRODUCTION_TABLE_ID`、`LARK_PRODUCTION_VIEW_ID` 和 `LARK_RESULT_FOLDER_TOKEN`。
4. 运行 `agent-config-probe --network` 与 `agent-smoke --production-bitable-read-only`。两者只会执行读取请求；后者最多读取一条需求文档，不会锁定、生成、创建结果表或写入记录。
5. 在本地页面确认扫描数量和抽样标题符合预期。扫描范围是“未开始、制作中、待修改”；临时测试时可通过 `LARK_INCLUDE_COMPLETED_FOR_TEST=true` 加入“已确认完成”。
6. 在首次真实交付前，确认结果文件夹已按飞书权限模型允许应用创建文件，并且应用可以为“需求制作人”添加结果表编辑权限。此权限不能由只读预检证明。
7. 首次通过本地审批并有产物可交付时，应用才会在结果文件夹下为该制作人创建一个结果表。每个制作人只复用一个表；每个需求只写入一行，重试会更新同一行。
8. 用户在结果表审核生成物。通过后由用户手动将生产表的“当前进度”改为已完成；应用不会自动切换该字段。

## 结果表字段

结果表按此顺序保存：`需求名称`、`需求附件`、`项目名称`、`发起人`、`需求制作人`、`结果`。其中前五列来自读取时的生产表快照，第六列只存生成产物附件。

## 本地锁的释放条件

领取任务后，本机锁会阻止同一条生产记录被重复领取。运行成功、运行失败、交付完成、超时终止或用户取消/删除本地运行后都会释放；生产表本身不受影响。
