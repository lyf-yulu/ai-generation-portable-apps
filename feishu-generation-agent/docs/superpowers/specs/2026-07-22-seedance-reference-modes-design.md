# Seedance 参考图模式设计

## 目标

让任务规划、审批编辑和 Seedance 提交使用同一套互斥参考图模式，避免把首尾帧与普通参考图混传给官方接口。

## 两种模式

### 首尾帧模式

- 只提交两张图片：一张 `first_frame`、一张 `last_frame`。
- 不提交 `reference_image`。
- 仅在需求明确要求首帧和尾帧、且没有额外视觉参考需求时自动选择。

### 多参考模式

- 所有提交图片均为 `reference_image`。
- 不提交 `first_frame` 或 `last_frame`。
- 需求含多张视觉参考时默认选择；即使文案提到首尾效果也一样。
- 规划提示词必须以文本描述首尾约束，例如开场状态接近第几张图、结尾状态接近第几张图，而不是使用首尾帧参数。

## 用户编辑

- 审批卡片展示“参考模式”选择。
- 切换到首尾帧模式时，用户必须保留且只保留首帧与尾帧两张图。
- 切换到多参考模式时，所有当前图片转换为普通参考图；用户可继续增删图片、调整顺序和编辑提示词。
- 服务端在批准前重新校验模式，拒绝混用并返回具体中文原因。

## 规划和执行边界

- 规划后的任务在进入审批前即规范化为合法模式。
- 前端编辑后的任务在批准时再次规范化/验证，保证手动修改不会生成非法请求。
- Seedance 适配器只接收已验证的角色组合；不得以放宽底层校验掩盖非法输入。

## 当前运行修复

“沉浸式切吐司”含首帧加三张普通参考图，按多参考模式处理：四张图均作为普通参考图，首尾意图通过提示词约束表达。
