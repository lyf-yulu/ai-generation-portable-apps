# Seedance 多模态参考素材设计

## 目标

在现有图片参考能力上，支持用户为 Seedance 视频任务添加参考视频和参考音频。图片继续走官方接口可接受的内嵌数据 URL；视频和音频在用户批准任务后上传到临时匿名公网托管，再把取得的 HTTPS 地址提交给官方 Seedance。

该能力面向本地部署的应用：客户端无需额外环境，服务端不依赖本机固定公网地址，并为未来替换为自有对象存储或公网中转保留边界。

## 范围与非目标

### 本次范围

- 多参考模式支持图片、视频、音频。
- 首尾帧模式仅支持一张首帧图片和一张尾帧图片。
- 审批页允许添加、预览、删除和重新排序参考素材。
- 仅在批准后、调用 Seedance 前上传本地视频或音频到临时公网托管。
- 提供明确的上传状态和失败提示，并可重新执行任务。

### 非目标

- 不将临时公网素材地址写回生产需求表。
- 不创建、修改或提交生产需求表中的状态。
- 不承诺第三方临时托管的文件撤回、保留期限或私密性。
- 不在本次引入自有对象存储、账户体系或稳定公网隧道。

## 方案选择

采用独立的 `PublicMediaHost` 适配层，首个实现为 Uguu 临时托管。

- **方案 A（采用）**：视频/音频上传到 Uguu，返回匿名 HTTPS URL。部署最简单，适合 AI 参考素材；代价是留存、删除和可用性不可控。
- **方案 B（未采用）**：通过临时 Cloudflare 公网隧道暴露本机 `/refmedia`。不重复上传，但依赖服务机持续公网可达，隧道重启还会使已提交 URL 失效。
- **方案 C（未采用）**：只接受用户提供的公网 URL。无需把文件传给第三方，但增加了用户操作成本。

## 数据和模式规则

### 统一参考素材

任务使用可扩展的参考素材集合，每一项包含本地媒体资产、角色、排序及可选的临时公网 URL。

| 素材类型 | 角色 | Seedance 提交形式 |
| --- | --- | --- |
| 图片 | `reference_image` | `image_url`，内嵌 data URL |
| 首帧图片 | `first_frame` | `image_url`，内嵌 data URL |
| 尾帧图片 | `last_frame` | `image_url`，内嵌 data URL |
| 视频 | `reference_video` | `video_url`，匿名 HTTPS URL |
| 音频 | `reference_audio` | `audio_url`，匿名 HTTPS URL |

兼容已有 `reference_images` 数据；迁移后仍可读取既有图片任务，不丢失审批草稿或历史运行。

### 参考模式

- **首尾帧模式**：只能保留 `first_frame` 与 `last_frame` 两张图片；不接受普通图片、视频、音频。
- **多参考模式**：可混合普通图片、参考视频和参考音频；不传首帧/尾帧参数。
- 已含视频或音频的任务，自动切换或保持为多参考模式。
- 需求同时提到首尾效果且给出多个参考素材时，默认多参考模式，并用提示词描述开场和结尾约束。

服务端会在计划阶段和批准阶段各验证一次，避免前端编辑造成混合模式请求。

## 执行链路

1. 用户在审批页添加图片、视频或音频；文件先只存放在本地运行目录。
2. 用户可在批准前调整参考模式、素材顺序、提示词和素材列表；此时不会外传视频或音频。
3. 用户批准后，执行器先验证任务及媒体格式。
4. 图片直接编码为 data URL；视频和音频经 `PublicMediaHost` 上传到 Uguu，取得匿名 HTTPS URL。
5. 执行器依据素材角色组装官方 Seedance 请求。参考视频、音频 URL 只在本次运行状态中缓存，便于同一次重试复用。
6. 若临时托管失败，任务不会提交给 Seedance，锁按既有失败规则释放，页面显示“临时托管失败”及可操作的原因。
7. Seedance 完成后，现有产物验证和飞书结果表写回流程保持不变。

## 前端交互

- 多参考模式下，统一的“添加参考素材”控件接受图片、视频、音频。
- 选择文件后立即显示文件名、类型和“正在添加/已添加/失败”状态，避免轮询刷新造成无反馈。
- 图片显示缩略图；视频显示可静音预览；音频显示播放器或音频文件标识。
- 每项素材可删除、调整顺序；图片可选普通参考图角色，首尾帧模式仅展示首帧与尾帧槽位。
- 执行前给出一次固定提示：参考视频和音频将上传至第三方临时公网托管，链接不可保证撤回或保留期限。

## 格式、体积与错误处理

- 首期接受常见视频格式：MP4、MOV、WebM；音频格式：MP3、WAV、M4A、AAC、OGG。
- 后端不只信任浏览器提供的 MIME 类型：按白名单、文件大小和基本文件签名验证；供应商最终校验仍以 Seedance 的响应为准。
- 单文件与总量上限从应用配置读取，不写死在前端；失败信息区分格式不支持、文件过大、网络超时、托管服务拒绝和 Seedance 拒绝。
- 临时托管 URL 必须是 HTTPS 且不包含鉴权信息；否则拒绝提交。

## 可替换边界

`PublicMediaHost` 只负责：接收本地文件、返回匿名公网 URL、报告可识别的失败原因。Seedance 适配器只消费该 URL，不关心 Uguu 的实现细节。

以后切换到自有对象存储、带过期控制的 CDN 或稳定公网中转时，只新增适配器和配置，不改审批界面、计划校验和 Seedance payload 组装规则。

## 验收标准

1. 多参考模式可添加图片、视频和音频；添加过程有即时状态反馈。
2. 首尾帧模式拒绝视频、音频和多余素材；混合素材会自动或明确要求切到多参考模式。
3. 批准前不会向第三方上传视频、音频。
4. 批准执行时，图片以 data URL、视频和音频以匿名 HTTPS URL 分别进入正确的 Seedance 字段和角色。
5. 临时托管失败不会消耗 Seedance 生成额度，且任务可重试。
6. 已有只含图片的任务、重跑任务和飞书结果写回不回归。
7. 单元、集成和前端状态测试覆盖格式校验、模式校验、托管失败和 payload 组装。
